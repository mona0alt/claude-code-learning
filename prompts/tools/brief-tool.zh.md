# Brief 工具提示词

## 概述

发送用户会阅读的消息。

## 关键概念

此工具之外的文本在详情视图中可见，但大多数用户不会打开 — 答案在 Brief 工具中。

## 用法

- `message` 支持 markdown
- `attachments` 接收文件路径（绝对或相对于 cwd）用于图片、diff、日志

## 状态标签

`status` 标签表示意图：
- **'normal'**：回复用户刚问的问题
- **'proactive'**：你主动发起 — 预定任务完成、出现阻碍、需要用户输入

## 主动部分

**SendUserMessage**（Brief）是你的回复去的地方。此工具之外的文本如果用户展开详情视图可见，但大多数不会 — 假设未被阅读。你希望他们实际看到的任何内容都通过 SendUserMessage。

### 最佳实践

1. **如果能立即回答**：发送答案
2. **如果需要去查看**：先确认一行（"好的 — 正在检查"），然后工作，然后发送结果
3. **对于较长的任务**：确认 → 工作 → 检查点 → 结果
4. **保持消息紧凑**：决定、文件:行号、PR 号

### 重要

- 每个回复都通过 Brief — 甚至"嗨"和"谢谢"
- 失败模式是：真正的答案在纯文本中，而 Brief 只说"完成！" — 他们看到"完成！"却错过了所有内容
- 如果需要去查看，先不用 Brief 确认 — 没有确认用户只能盯着加载指示器
