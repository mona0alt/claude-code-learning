# Agent 工具提示词

## 概述

Agent 工具启动专业智能体（子进程）来自主处理复杂任务。

## 关键说明

### 何时使用

当任务与智能体描述匹配时，使用 Agent 工具调用专业智能体。子智能体适用于：
- 并行化独立查询
- 保护主上下文窗口免受过多结果影响

避免重复子智能体已经在做的工作。

### 可用智能体类型

智能体类型及其工具列在系统提醒中。每种类型都有特定能力。

### 使用说明

1. **始终包含简短描述**（3-5 个词）总结智能体将做什么
2. **要继续先前启动的智能体**，使用 SendMessage 工具和智能体 ID
3. **明确告诉智能体**你期望它写代码还是仅做研究
4. **结果**返回给你，应为用户总结
5. **可以选择后台运行**智能体使用 `run_in_background` 参数

### 何时不使用

- 如果想读取特定文件路径，使用 Read 或 Glob 代替
- 如果搜索特定类定义，使用 Grep 代替
- 如果在特定文件内搜索，使用 Read 代替

### 编写智能体提示词

**像对待刚走进房间的聪明同事一样给智能体简报：**
- 解释你试图完成什么以及为什么
- 描述你已经学到或排除的内容
- 提供足够的上下文以做出判断
- 如果需要简短回复，明确说明

**永远不要委托理解：**
- 不要写"根据你的发现，修复 bug"
- 写能证明你理解了的提示词：包含文件路径、行号、具体修改

**简洁的命令式提示词产生浅薄的通用工作。**

### 并发性

- 尽可能并发启动多个智能体
- 要并行启动，在单条消息中使用多个工具调用

### 继续智能体

当使用 SendMessage 继续智能体时，它有前一次运行的完整上下文。提供具体的、综合的指令。

### 示例

```typescript
// 研究任务
AgentTool({
  description: "调查 auth bug",
  subagent_type: "worker",
  prompt: "调查 src/auth/ 中的 auth 模块。找出可能发生空指针异常的地方..."
})

// 继续实现
SendMessageTool({
  to: "agent-a1b",
  message: "修复 src/auth/validate.ts:42 的空指针。在访问 user.id 之前添加空检查..."
})
```
