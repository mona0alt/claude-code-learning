# Task 工具提示词

## TaskCreate 工具

在任务列表中创建新任务。

### 何时使用

- **复杂多步骤任务** - 需要 3 个或更多独立步骤或操作
- **非平凡的复杂任务** - 需要仔细规划或多个操作
- **计划模式** - 创建任务列表来跟踪工作
- **用户明确要求**待办列表
- **用户提供多个任务** - 编号或逗号分隔的列表
- **收到新指令后** - 将需求捕获为任务
- **开始工作时** - 在开始前标记为 in_progress
- **完成任务后** - 标记为已完成并添加实现过程中发现的后续任务

### 何时不使用

- 只有单一、直接的任务
- 任务琐碎且跟踪没有好处
- 任务可以在不到 3 个简单步骤中完成
- 任务纯粹是对话式或信息性的

### 任务字段

- **subject**: 简洁、可操作的任务标题，使用祈使句式（例如："修复登录流程中的认证 bug"）
- **description**: 需要完成的事情
- **activeForm**（可选）：任务进行时微调显示的现在分词形式（例如："正在修复认证 bug"）

### 技巧

- 创建任务时使用描述结果的可操作的具体标题
- 如需要，使用 TaskUpdate 设置依赖关系（blocks/blockedBy）
- 先检查 TaskList 避免创建重复任务

---

## TaskUpdate 工具

更新任务列表中的任务。

### 可更新的内容

- **status**: `pending` → `in_progress` → `completed`
- **subject**: 更改任务标题
- **description**: 更改任务描述
- **activeForm**: 更改微调文本
- **owner**: 为任务分配负责人
- **blocks**: 标记此任务阻止的任务
- **blockedBy**: 标记必须先完成的任务

### 状态流程

状态推进：`pending` → `in_progress` → `completed`

---

## TaskList 工具

列出任务列表中的所有任务。

### 显示内容

- 任务 ID
- 主题/简短描述
- 状态
- 负责人
- blockedBy 关系

### 用法

用于：
- 查看可用任务
- 检查整体项目进度
- 查找被阻止的任务
- 避免创建重复任务

---

## TaskGet 工具

通过 ID 获取任务。

### 返回内容

- 完整任务详情包括描述
- 状态
- 此任务阻止什么以及什么阻止它

### 用法

在开始任务前使用以获取完整需求。

---

## TaskStop 工具

停止运行中的后台任务。

### 何时使用

- 任务启动错误时
- 任务不再需要时
- 任务需要取消时
