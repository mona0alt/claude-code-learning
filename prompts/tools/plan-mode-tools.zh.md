# 计划模式工具

## EnterPlanMode 工具

当你即将开始**非平凡的实现任务**时使用此工具。在写代码之前获得用户确认可以防止浪费精力。

### 何时使用

以下实现任务**使用 EnterPlanMode**：

1. **新功能实现** - 添加有意义的新功能
2. **多种有效方法** - 任务有多种解决方式
3. **代码修改** - 影响现有行为的更改
4. **架构决策** - 在模式或技术之间选择
5. **多文件更改** - 任务可能涉及超过 2-3 个文件
6. **需求不明确** - 需要探索才能理解完整范围
7. **用户偏好重要** - 实现可能有多种方向

### 何时不使用

简单任务跳过：
- 单行修复（拼写错误、明显 bug）
- 添加有明确需求的单个函数
- 有非常具体详细指令的任务
- 纯研究/探索任务（使用 Agent 工具）

### 计划模式中会发生什么

1. 使用 Glob、Grep、Read 探索代码库
2. 理解现有模式和架构
3. 设计实现方法
4. 向用户展示计划以获得批准
5. 如需澄清，使用 AskUserQuestion
6. 准备好实现时退出计划模式

---

## ExitPlanMode 工具

当你写完计划并准备好用户批准时使用此工具。

### 工作原理

- 你应该已经将计划写入计划文件
- 此工具不接受计划内容作为参数
- 只是标记你完成计划
- 用户将看到你的计划文件内容

### 何时不使用

只是收集信息的研究任务 — 不要使用此工具。

### 使用前

确保你的计划完整：
- 首先对未解决的问题使用 AskUserQuestion
- 完成后，使用此工具请求批准

### 重要

不要使用 AskUserQuestion 问"这个计划可以吗？" — 那正是 ExitPlanMode 的作用。
