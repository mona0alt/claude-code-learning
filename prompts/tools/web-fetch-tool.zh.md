# WebFetch 工具提示词

## 概述

从指定 URL 获取内容并使用 AI 模型处理。

## 工作原理

1. 接收 URL 和提示词作为输入
2. 获取 URL 内容，将 HTML 转换为 markdown
3. 使用快速模型处理内容
4. 返回模型关于内容的响应

## 使用说明

- 如果有 MCP 提供的 web fetch 工具，**优先使用**（限制更少）
- URL 必须是完整有效的
- HTTP URL 自动升级为 HTTPS
- 内容非常大时可能会被摘要
- 15 分钟缓存以加快重复访问
- **对于 GitHub URL**，优先使用 Bash 的 `gh` CLI

## 提示词中应包含什么

描述你想从页面提取什么信息。

## 重定向处理

如果 URL 重定向到不同主机，会告知你重定向 URL。使用重定向 URL 发新的 WebFetch 请求。

## 重要

- 此工具是**只读**的，不会修改任何文件
